## Phần 1: Chuẩn bị môi trường

### 1.1 Cài PostgreSQL Client 17

```bash
# Thêm repo PostgreSQL chính thức
sudo mkdir -p /usr/share/keyrings
wget -qO - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo gpg --dearmor -o /usr/share/keyrings/postgresql-archive-keyring.gpg

echo "deb [signed-by=/usr/share/keyrings/postgresql-archive-keyring.gpg] http://apt.postgresql.org/pub/repos/apt noble-pgdg main" | sudo tee /etc/apt/sources.list.d/pgdg.list

sudo apt update
sudo apt install postgresql-client-17
```

Kiểm tra:
```bash
psql --version
```

### 1.2 Tạo Virtual Environment (nếu chưa có)

```bash
cd backend
python3 -m venv .venv
```

### 1.3 Cài Python Dependencies

```bash
pip install django djangorestframework pydantic python-dotenv psycopg2-binary
```

## Phần 2: Chạy Cloud SQL Proxy (Docker)

### 2.1 Chuẩn bị Service Account Key

- Đảm bảo file service account JSON tồn tại: cao-pizza-firebase-adminsdk-fbsvc-8a1650077e.json
- Gán IAM roles cho service account (gcloud CLI):
  - `cloudsql.client` (Cloud SQL Client)
  - `editor` hoặc `storage.admin` (nếu dùng Firebase Storage)

### 2.2 Khởi động Cloud SQL Proxy Container

```bash
# Xác định Cloud SQL instance connection string:
# Format: project-id:region:instance-name
# Ví dụ: cao-pizza:us-central1:postgres-db

docker run -d \
  --name cloudsql-proxy \
  -p 5432:5432 \
  gcr.io/cloudsql-docker/gce-proxy:latest \
  /cloud_sql_proxy \
  -instances=cao-pizza:us-central1:postgres-db=tcp:0.0.0.0:5432 \
  -credential_file=/config/key.json \
  -use_dns=false \
  -ip=0.0.0.0
```

Kiểm tra trạng thái:
```bash
docker ps --format "table {{.ID}}\t{{.Names}}\t{{.Ports}}"
docker logs cloudsql-proxy --tail 50
```

Kết quả mong đợi: "Listening on 0.0.0.0:5432" hoặc "Ready for new connections"

### 2.3 Cấu hình Database User Password (gcloud CLI)

```bash
gcloud sql users set-password postgres \
  --instance=postgres-db \
  --password=Titeuf1234
```

```bash
DB_NAME=pizza_ordering_db
```

## Troubleshooting

### Connection refused (port 5432)

```bash
# Kiểm tra proxy chạy chưa
docker ps | grep cloudsql-proxy

# Nếu chưa, start lại:
docker start cloudsql-proxy

# Hoặc check logs:
docker logs cloudsql-proxy
```

### Permission denied / 403

- Đảm bảo service account có role `Cloud SQL Client`.
- Ngược lại nếu dùng Firebase Storage, thêm role `Storage Admin`.

```bash
gcloud projects add-iam-policy-binding cao-pizza \
  --member=serviceAccount:SERVICE_ACCOUNT_EMAIL \
  --role=roles/cloudsql.client
```

### ModuleNotFoundError (import)

```bash
# Chắc chắn virtualenv active
source .venv/bin/activate
pip list  # Kiểm tra packages
```

### Database does not exist

- Chắc chắn đã tạo database `pizza_ordering_db`.
- Kiểm tra .env có giá trị `DB_NAME=pizza_ordering_db`.

```sql
-- Trong psql
SELECT datname FROM pg_database WHERE datname='pizza_ordering_db';
```
